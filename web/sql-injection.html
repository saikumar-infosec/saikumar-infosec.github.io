📌 SQL Injection (SQLi) – Basic to Advanced

🧠 Description

What is SQL Injection?

SQL Injection is a code injection technique that allows attackers to interfere with the queries an application makes to its database. It occurs when user input is improperly sanitized before being included in SQL queries.

Types of SQL Injection:

  • Error-Based SQLi: Relies on error messages to extract information
  • Union-Based SQLi: Uses UNION operator to combine results
  • Boolean-Based Blind SQLi: Infers information based on true/false responses
  • Time-Based Blind SQLi: Uses time delays to infer information
  • Out-of-Band SQLi: Uses DNS or HTTP requests to exfiltrate data
  • Second-Order SQLi: Payloads stored and executed later

Security principle violated: Input Validation

🏷️ Classification

  • Type: Injection
  • OWASP: A03:2021 – Injection
  • CWE: CWE-89: Improper Neutralization of Special Elements used in an SQL Command
  • CAPEC: CAPEC-66: SQL Injection

🎯 Attack Surface (Where to Test)

  • ✅ URL parameters (?id=, ?user=, ?page=)
  • ✅ POST body (JSON, form-data, XML, multipart/form-data)
  • ✅ Headers (User-Agent, Referer, X-Forwarded-For, Cookie)
  • ✅ Cookies / Sessions
  • ✅ File uploads (filename injection)
  • ✅ API endpoints (REST, GraphQL)
  • ✅ Order By / Sort parameters
  • ✅ JSON properties, XML nodes

🔍 Detection Methodology

1. Static Analysis (Source Code)

// Vulnerable code pattern - String concatenation
String query = "SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'";
// Vulnerable PHP
$query = "SELECT * FROM users WHERE id = " . $_GET['id'];

2. Dynamic Analysis (Burp Suite / Manual)

  • Intercept request and inject single quote (')
  • Inject double quote ("), backtick (`)
  • Inject ' OR '1'='1, ' AND SLEEP(5)--
  • Observe error messages, response changes, time delays

Tools: Burp Suite (Repeater, Intruder), SQLmap, OWASP ZAP, FFUF, Nuclei

⚙️ Exploitation Steps

  1. Identify vulnerable parameter (test with ', ", ')).
  2. Determine database type (MySQL, PostgreSQL, MSSQL, Oracle) using error messages or version payloads.
  3. Use UNION attacks to extract data.
  4. Use boolean/time‑based techniques for blind injection.
  5. Extract table names, column names, and data.
  6. Escalate to file read/write or OS command execution (if applicable).
  7. Use SQLmap for automated exploitation.

🛡️ Mitigation

✅ Parameterized Queries (Prepared Statements): Always use prepared statements with bound parameters.
✅ Input Validation: Whitelist allowed characters and validate data types.
✅ Stored Procedures: Use stored procedures with limited privileges.
✅ Least Privilege: Database accounts should have minimal required permissions.
✅ WAF: Deploy Web Application Firewall with SQL injection rules.
✅ Disable Dangerous Functions: Disable xp_cmdshell, UDF, OLE automation when not needed. 
// Secure implementation (Java - PreparedStatement)
String query = "SELECT * FROM users WHERE id = ?";
PreparedStatement stmt = conn.prepareStatement(query);
stmt.setInt(1, Integer.parseInt(userId));

// Python - Parameterized query
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))

// PHP - PDO
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute(['id' => $userId]);

⚠️ Risk / Impact

Severity: Critical

Impact:
  • Data breach (usernames, passwords, PII, financial data)
  • Authentication bypass
  • Data manipulation (insert, update, delete)
  • Remote Code Execution (via xp_cmdshell, UDF)
  • Denial of Service (delete tables, drop databases)
