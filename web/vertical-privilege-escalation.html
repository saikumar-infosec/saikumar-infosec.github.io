📌 Vertical Privilege Escalation (User to Admin)

🧠 Description

What is the vulnerability?

A regular user can perform actions reserved for administrators by manipulating parameters, accessing hidden endpoints, or exploiting flawed access controls. This leads to full application compromise.

Why does it occur?

Missing or improper authorization checks on server‑side, reliance on client‑side role checks, or insecure direct object references.

Security principle violated: Broken Access Control

Real-world analogy: A museum visitor walking into the security control room because the door is unlocked – no special key required.

🏷️ Classification

  • Type: Broken Access Control
  • OWASP: A01:2021 – Broken Access Control
  • CWE: CWE-269: Improper Privilege Management

🎯 Attack Surface

  • ✅ Hidden admin endpoints (`/admin`, `/api/admin`)
  • ✅ Role parameters in JWT, cookies, or POST bodies
  • ✅ Missing authorization checks on API endpoints
  • ✅ Parameter tampering (`?role=admin`)

🔍 Detection Methodology

  • Enumerate admin paths (`/admin`, `/administrator`, `/dashboard`, `/api/admin`).
  • Modify JWT payload: change `"role":"user"` to `"role":"admin"` (if secret is weak or none algorithm).
  • Intercept requests and add `isAdmin=true` or similar parameters.

💣 Basic Payloads

# Original request
GET /api/user/profile

# Modified to access admin endpoint
GET /api/admin/users

🚀 Advanced Bypass

# Add role parameter to POST body
{"username":"user", "role":"admin"}

# Change JWT token claim
{"alg":"HS256","typ":"JWT"} -> {"alg":"none","typ":"JWT"}

🧪 Test Cases (STRICT)

✔ Must be reproducible

⚙️ Exploitation Steps

  1. Identify admin endpoints (dirb, ffuf, or review JS files).
  2. Attempt to access them using a low‑privilege session.
  3. If successful, document impact (data exposure, account creation, configuration changes).
  4. Escalate to full admin control.

🛡️ Mitigation

✅ Implement role‑based access control (RBAC) on every endpoint.
✅ Never rely on client‑side role checks.
✅ Use server‑side session with role stored securely.
✅ Regularly audit API endpoints for missing authorization.
✅ Use automated tools like Autorize or Authz to test privilege escalation.

🔓 Mitigation Bypass

  • If authorization is only checked on some endpoints, attackers may find overlooked ones.
  • If admin endpoints are hidden but not protected, directory fuzzing can discover them.

⚠️ Risk / Impact

Severity: Critical

Impact: Full administrative control, data breach, account takeover, system compromise.

📎 Proof of Concept (PoC)

# Direct admin access via URL
curl https://target.com/admin/users -H "Cookie: session=user_session"
