📌 Weak Cryptographic Algorithm

🧠 Description

What is the vulnerability?

Use of outdated or broken cryptographic algorithms (MD5, SHA-1, DES, RC4) for hashing or encryption. These are vulnerable to collision attacks, brute force, or other cryptanalysis techniques.

Why does it occur?

Legacy code, lack of security awareness, or failure to update dependencies.

Security principle violated: Cryptographic Failure

Real-world analogy: Using a wooden lock on a bank vault – it may look like a lock but offers no real security.

🏷️ Classification

  • Type: Cryptographic Failure
  • OWASP: A02:2021 – Cryptographic Failures
  • CWE: CWE-327: Use of a Broken or Risky Cryptographic Algorithm

🎯 Attack Surface

  • ✅ Password hashing (MD5, SHA-1)
  • ✅ SSL/TLS cipher suites (RC4, DES)
  • ✅ JWT algorithms (HS256 with weak secret, none)
  • ✅ Session token generation

🔍 Detection Methodology

  • Extract password hashes from database – identify hash type using `hashid`.
  • Use SSL Labs to check for weak ciphers.
  • Analyze JWT headers for `alg: HS256` with weak secrets.

💣 Basic Payloads (Weak Hashes)

# MD5 of "password"
5f4dcc3b5aa765d61d8327deb882cf99

# SHA-1 of "password"
5baa61e4c9b93f3f0682250b6cf8331b7ee68fd8

🚀 Advanced Exploitation

# Crack MD5 with hashcat
hashcat -a 0 -m 0 hash.txt rockyou.txt

# Downgrade TLS to use RC4
curl --ciphers RC4-SHA https://target.com

⚙️ Exploitation Steps

  1. Extract hash from database.
  2. Identify algorithm (hashid).
  3. Crack using rainbow tables or brute force (hashcat).
  4. Use cracked password to authenticate.

🛡️ Mitigation

✅ Use strong hashing: bcrypt, Argon2, PBKDF2.
✅ Disable weak TLS ciphers (RC4, DES, 3DES).
✅ Use AES-256 for symmetric encryption.
✅ For JWT, enforce strong algorithms (RS256/ES256) with long keys.

🔓 Mitigation Bypass

  • If MD5 is used with a salt, it's still weak – attackers can precompute with the salt.
  • If weak ciphers are not disabled, attackers can force downgrade attacks.

⚠️ Risk / Impact

Severity: High

Impact: Credential cracking, data decryption, session forgery.

📎 Proof of Concept (PoC)

# Identify hash
hashid -m 5f4dcc3b5aa765d61d8327deb882cf99

# Crack with hashcat
hashcat -a 0 -m 0 hash.txt rockyou.txt
