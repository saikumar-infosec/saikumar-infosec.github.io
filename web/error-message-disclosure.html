📌 Sensitive Data in Error Messages

🧠 Description

What is the vulnerability?

Error messages (stack traces, SQL errors, path disclosures) reveal internal details about the application, database, or server configuration. Attackers use this information to refine further attacks.

Why does it occur?

Developers leave debug mode enabled in production or fail to implement proper exception handling and custom error pages.

Security principle violated: Information Disclosure

Real-world analogy: A locked door that shows a detailed blueprint of the building when you knock – attackers learn exactly where to strike.

🏷️ Classification

  • Type: Information Disclosure
  • OWASP: A04:2021 – Insecure Design / A05:2021 – Security Misconfiguration
  • CWE: CWE-209: Generation of Error Message Containing Sensitive Information

🎯 Attack Surface

  • ✅ Invalid input triggers SQL errors
  • ✅ Missing files cause path disclosure
  • ✅ Debug mode enabled in production
  • ✅ API endpoints returning verbose errors

🔍 Detection Methodology

  • Inject unexpected characters (`'`, `\`, `%`, `
