📌 Vulnerable WordPress Plugins & Themes

🧠 Description

What is the vulnerability?

WordPress sites using outdated or poorly coded plugins/themes expose SQLi, XSS, RCE, and authentication bypass vulnerabilities. Attackers can exploit these to take over the site, deface it, or steal data.

Why does it occur?

Site owners fail to update plugins/themes, use nulled (pirated) versions, or install plugins from untrusted sources.

Security principle violated: Vulnerable Components

Real-world analogy: Installing a third‑party lock on your door without checking if it has a master key.

🏷️ Classification

  • Type: Vulnerable Components
  • OWASP: A06:2021 – Vulnerable and Outdated Components
  • CWE: CWE-1104: Use of Unmaintained Third Party Components

🎯 Attack Surface

  • ✅ /wp-content/plugins/ (plugin directories)
  • ✅ /wp-content/themes/ (theme directories)
  • ✅ readme.txt files revealing versions
  • ✅ Admin Ajax endpoints

🔍 Detection Methodology

# WPScan – WordPress vulnerability scanner
wpscan --url https://target.com --enumerate vp,vt

# Nuclei WordPress templates
nuclei -t technologies/wordpress/ -target https://target.com

# Manual: Check plugin readme
https://target.com/wp-content/plugins/vulnerable-plugin/readme.txt

💣 Basic Payloads (Example: Plugin RCE)

# Vulnerable plugin with direct file inclusion
https://target.com/wp-content/plugins/vulnerable-plugin/ajax.php?cmd=id

# SQLi in plugin parameter
https://target.com/index.php?plugin_id=1 AND 1=0 UNION SELECT 1,2,3

🚀 Advanced Exploitation

# Use wpscan to get exploit details
wpscan --url https://target.com --enumerate vp --api-token YOUR_TOKEN

# Metasploit module for specific plugin
use exploit/unix/webapp/wp_plugin_upload_rce

⚙️ Exploitation Steps

  1. Enumerate installed plugins/themes using wpscan or directory fuzzing.
  2. Identify vulnerable versions from readme.txt or version numbers.
  3. Search for public exploits (Exploit‑DB, WPVulnDB).
  4. Execute exploit to gain admin access or RCE.

🛡️ Mitigation

✅ Keep WordPress core, plugins, themes updated.
✅ Remove unused plugins/themes immediately.
✅ Use a security plugin (Wordfence, Sucuri).
✅ Regularly run wpscan or similar scanners.
✅ Avoid nulled (pirated) plugins/themes.
✅ Implement Web Application Firewall (WAF).

🔓 Mitigation Bypass

  • If auto‑update is enabled but the plugin's update server is compromised.
  • If a zero‑day vulnerability exists in a popular plugin.

⚠️ Risk / Impact

Severity: Critical

Impact: Full site takeover, data breach, malware distribution, SEO spam.
