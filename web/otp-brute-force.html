📌 OTP Brute Force (Missing Rate Limiting)

🧠 Description

What is the vulnerability?

One‑Time Password (OTP) verification endpoint lacks rate limiting or account lockout. Attackers can brute force 4‑6 digit OTPs (10,000 to 1,000,000 combinations) using automated requests.

Why does it occur?

Developers assume OTPs are secure by design and forget to implement brute force protection.

Security principle violated: Weak Authentication / Rate Limiting

Real-world analogy: A padlock with a 4‑digit code that doesn't lock after wrong attempts – you can try all 10,000 combinations.

🏷️ Classification

  • Type: Authentication Bypass
  • OWASP: A07:2021 – Identification and Authentication Failures
  • CWE: CWE-307: Improper Restriction of Excessive Authentication Attempts

🎯 Attack Surface

  • ✅ OTP verification endpoints (2FA, password reset, email verification)
  • ✅ Mobile OTP login flows
  • ✅ API endpoints that accept OTP codes
  • ✅ Web forms with OTP input

🔍 Detection Methodology

  • Request an OTP for your account.
  • Use Burp Intruder or a script to send multiple OTP guesses (e.g., 0000 to 9999).
  • If you find the correct OTP within a few thousand attempts, rate limiting is missing.
  • Check if account locks after X failures or if there's a CAPTCHA.

💣 Basic Payloads

# Burp Intruder payload: 0000 to 9999
POST /verify-otp
otp=§0000§&phone=+1234567890

# Python script
import requests
for otp in range(10000):
    r = requests.post('https://target.com/verify', data={'otp': f'{otp:04d}'})
    if 'success' in r.text:
        print(f'Found OTP: {otp:04d}')
        break

🚀 Advanced Exploitation

# Distributed brute force across multiple IPs (if IP‑based rate limiting exists)
# Use proxychains or rotating VPNs

# Time‑based attacks if OTP is valid for long period (e.g., 10 minutes)
# 10,000 attempts / 10 minutes = 16.6 attempts/second – feasible

⚙️ Exploitation Steps

  1. Trigger OTP generation for a target account (phone/email).
  2. Capture the OTP verification request.
  3. Use Burp Intruder or custom script to iterate through all possible OTP values.
  4. When the correct OTP is found, the response will indicate success.
  5. Log in as the victim or reset their password.

🛡️ Mitigation

✅ Implement rate limiting (e.g., 5 attempts per minute, 20 per hour).
✅ Lock out after 10 failed attempts (temporary or permanent).
✅ Use CAPTCHA after 3 failures.
✅ Increase OTP length to 8 digits (100 million combos).
✅ Use time‑based OTP (TOTP) with short validity (30–60 seconds).
✅ Implement exponential backoff for failed attempts.

🔓 Mitigation Bypass

  • If rate limiting is IP‑based, attackers can use IP rotation or botnets.
  • If lockout is based on account, attackers can target multiple accounts with same OTP.

⚠️ Risk / Impact

Severity: High

Impact: Account takeover, bypass 2FA, password reset compromise, financial fraud.
