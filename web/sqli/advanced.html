📌 Advanced SQL Injection (SQLi)

🧠 Description

Advanced SQL injection techniques go beyond basic error‑based or union‑based attacks. They leverage database‑specific functions, encoding, obfuscation, and even server‑side features (e.g., OLE automation, UDFs, Java procedures) to extract data, escalate privileges, or achieve remote code execution.

This guide covers error‑based, union‑based, blind, WAF bypass, stacked queries, DNS exfiltration, and database‑specific exploitation (MySQL, PostgreSQL, MSSQL, Oracle, SQLite).

🏷️ Classification

  • Type: Injection / Advanced Exploitation
  • OWASP: A03:2021 – Injection
  • CWE: CWE-89, CWE-90, CWE-94

🎯 Attack Surface

  • ✅ All standard injection points (URL, POST, headers, cookies)
  • ✅ JSON/XML parameters
  • ✅ Order by / sort parameters
  • ✅ File upload filenames (second‑order injection)

🔍 Detection Methodology

1. Advanced Error‑Based

-- MySQL duplicate entry error (floor/rand)
' AND (SELECT 1 FROM (SELECT COUNT(*), CONCAT((SELECT version()), 0x3a, FLOOR(RAND(0)*2)) x FROM information_schema.tables GROUP BY x) y) --

2. Time‑Based Blind with Conditional Responses

-- MySQL: if database name length > 5, sleep 5 seconds
' AND IF((SELECT LENGTH(database())>5), SLEEP(5), 0) --

3. Bitwise Boolean Blind

-- Check first ASCII bit
' AND IF((SELECT ASCII(SUBSTRING((SELECT database()),1,1))) & 1, SLEEP(5), 0) --

⚙️ Exploitation Steps

  1. Identify injection point and database type (error‑based fingerprinting).
  2. Use advanced error payloads to extract version, database name, user.
  3. If error‑based not possible, use time‑based boolean with binary search.
  4. Apply obfuscation to bypass WAF (encoding, comments, case mixing).
  5. Extract schema, tables, columns, and data using union or stacked queries.
  6. Escalate to file read/write (LOAD_FILE, INTO OUTFILE) or command execution (xp_cmdshell, UDF).

🛡️ Mitigation

✅ Parameterized queries (prepared statements)
✅ Strict input validation (allowlist)
✅ WAF with advanced rule sets (e.g., ModSecurity, Cloudflare)
✅ Least privilege database accounts
✅ Disable dangerous functions (xp_cmdshell, UDF, OLE automation)
✅ Regular security updates and database hardening

⚠️ Risk / Impact

Severity: Critical

Impact: Full database compromise, arbitrary file read/write, remote code execution, data exfiltration, privilege escalation.
