📌 Full Credit Card Number Exposure

🧠 Description

What is the vulnerability?

The application stores or displays full credit card numbers (PAN) in plaintext, violating PCI-DSS requirements. Attackers with database access or XSS can steal complete payment information.

Why does it occur?

Poor compliance awareness, improper tokenization, or insecure storage practices.

Security principle violated: Sensitive Data Exposure

Real-world analogy: Writing your credit card number on the outside of an envelope – anyone who sees it can use it.

🏷️ Classification

  • Type: Sensitive Data Exposure
  • OWASP: A02:2021 – Cryptographic Failures / A04:2021 – Insecure Design
  • CWE: CWE-312: Cleartext Storage of Sensitive Information

🎯 Attack Surface

  • ✅ Payment confirmation pages
  • ✅ Admin order panels
  • ✅ API responses containing full card numbers
  • ✅ Database backups

🔍 Detection Methodology

  • Complete a test transaction and inspect response bodies for 16-digit numbers.
  • Search database for patterns matching credit card numbers (Luhn algorithm).
  • Check admin interfaces for exposed card data.

💣 Basic Example

{"card_number": "4111111111111111", "expiry": "12/28", "cvv": "123"}

🚀 Advanced Exploitation

# SQL injection to dump credit cards
' UNION SELECT card_number, expiry FROM payments --

🧪 Test Cases

⚙️ Exploitation Steps

  1. Gain access to database via SQL injection or leaked backup.
  2. Dump the payments or orders table.
  3. Use stolen credit card numbers for fraudulent transactions.

🛡️ Mitigation

✅ Never store full PAN after authorization (PCI-DSS requirement).
✅ Tokenize card numbers via payment gateway.
✅ Display only last 4 digits.
✅ Encrypt stored card data (AES-256).
✅ Regular PCI-DSS audits.

🔓 Mitigation Bypass

  • If encryption keys are stored alongside the data, encryption provides little protection.
  • If tokenization is used but tokens are reversible, it's equivalent to plaintext.

⚠️ Risk / Impact

Severity: Critical

Impact: Financial fraud, regulatory fines (PCI-DSS, GDPR), reputational damage.
