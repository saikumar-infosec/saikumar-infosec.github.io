📌 Weak Password Policy

🧠 Description

What is a Weak Password Policy?

A weak password policy allows users to set easily guessable passwords (e.g., "password", "123456", "admin"). Without complexity requirements, length limits, or account lockout mechanisms, attackers can perform brute‑force or credential stuffing attacks to compromise accounts.

Why does it occur?

Poorly configured password policies – missing length checks, no character type requirements, and no protection against common passwords.

Security principle violated: Authentication / Secure Configuration

Real-world analogy: Like a bank vault that accepts the combination "1234" – it technically works, but offers no real security.

🏷️ Classification

  • Type: Weak Authentication
  • OWASP: A07:2021 – Identification and Authentication Failures
  • CWE: CWE-521: Weak Password Requirements

🎯 Attack Surface (Where to Test)

  • ✅ Registration page – check allowed passwords
  • ✅ Change password functionality
  • ✅ Password reset flows
  • ✅ Login page for spraying attempts

🔍 Detection Methodology

1. Static Analysis (Source Code)

// Vulnerable password validation (PHP)
if (strlen($password) < 6) { // too short
    // accept password
}

2. Dynamic Analysis (Burp Suite / Manual)

  • Attempt to register with weak passwords: "password", "123456", "qwerty", "admin", "letmein"
  • Check minimum length (usually should be ≥ 8)
  • Check complexity requirements (uppercase, lowercase, number, special character)
  • Use Burp Intruder with a list of common passwords during registration

✔ Must be reproducible

⚙️ Exploitation Steps

  1. Identify registration or password change functionality.
  2. Test with common weak passwords to see what is allowed.
  3. If weak passwords are accepted, use credential stuffing with breached credentials.
  4. Password spray common passwords across many accounts to avoid lockout.

🛡️ Mitigation

✅ Enforce strong password policies: Minimum 12 characters, require uppercase, lowercase, numbers, and special characters.
✅ Block common passwords: Maintain a blacklist of known weak passwords (e.g., rockyou.txt).
✅ Implement account lockout after failed attempts.
✅ Use multi‑factor authentication (MFA).
✅ Regular password expiry (but not too frequent).
✅ Check passwords against breached credential databases (HaveIBeenPwned API).

🔓 Mitigation Bypass

  • If only length is enforced (e.g., 8 chars), use common short passwords like "password".
  • If complexity is required but predictable, use "Password1!" – still weak.
  • If no lockout, perform large‑scale password spraying.

⚠️ Risk / Impact

Severity: Medium to High

Impact: Account takeover, data theft, financial fraud, reputation damage.
