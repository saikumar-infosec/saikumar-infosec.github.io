Critical Severity

📌 MIME Type Bypass

🧠 Description

What is MIME Type Bypass?

The server checks only the Content-Type header. Attackers can modify the MIME type to bypass validation.

Why does it occur?

Server trusts client-supplied Content-Type header instead of validating actual file content.

Security principle violated: Input Validation / Trust Boundary

🏷️ Classification

  • Type: MIME Type Bypass
  • OWASP: A05:2021 – Security Misconfiguration
  • CWE: CWE-434: Unrestricted Upload

🎯 Attack Surface

  • ✅ File upload endpoints with Content-Type validation only
  • ✅ API endpoints accepting multipart/form-data
  • ✅ Applications using getMimeType() without content verification

🔍 Detection Methodology

1. Static Analysis (Source Code)

// Vulnerable code - only checks Content-Type header
if ($_FILES['file']['type'] == 'image/jpeg') {
    move_uploaded_file(...);
}

2. Dynamic Analysis (Burp Suite / Manual)

  • Intercept upload request in Burp Suite
  • Modify Content-Type header to allowed value
  • Observe if upload succeeds

💣 Basic Payloads

# Change Content-Type header to:
Content-Type: image/jpeg
Content-Type: image/png
Content-Type: application/pdf
Content-Type: text/plain

🚀 Advanced Payloads

# Using curl
curl -F "file=@shell.php;type=image/jpeg" https://target.com/upload

# Multiple content-type headers
Content-Type: image/jpeg
Content-Type: application/x-php

🧪 Test Cases (STRICT)

StepActionExpectedActual
1Upload shell.php with Content-Type: image/jpegRejectedUpload successful
2Access uploaded fileNo executionPHP code executes

⚙️ Exploitation Steps

  1. Intercept file upload request in Burp Suite
  2. Change Content-Type header to allowed type (image/jpeg)
  3. Forward request and check if upload succeeded
  4. Access uploaded shell to confirm execution

🛡️ Mitigation

✅ Validate File Content: Check magic bytes/file signatures, not Content-Type header
✅ Use Server-Side Validation: Never trust client-supplied headers
✅ Implement File Type Detection: Use libraries like file, libmagic, or finfo 
# PHP - Validate actual file type
$finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime_type = finfo_file($finfo, $_FILES['file']['tmp_name']);
if (!in_array($mime_type, ['image/jpeg', 'image/png'])) {
    die('Invalid file type');
}

⚠️ Risk / Impact

Severity: High to Critical

Impact: Remote Code Execution, Server Compromise
