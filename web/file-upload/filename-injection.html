📌 File Name Injection

🧠 Description

What is File Name Injection?

Injecting malicious content via file name that gets reflected in responses or logs.

Why does it occur?

Application reflects filename without proper encoding or sanitization.

Security principle violated: Output Encoding

🏷️ Classification

  • Type: Injection
  • CWE: CWE-79, CWE-93, CWE-116

🎯 Attack Surface

  • ✅ File download pages displaying filenames
  • ✅ Admin panels listing uploaded files
  • ✅ Log files recording uploaded filenames

💣 Basic Payloads

# XSS via filename
<script>alert(1)</script>.jpg
<img src=x onerror=alert(1)>.png

# Command injection via filename
; rm -rf / ;.jpg
| whoami .png
$(id).gif

# Log injection
test.jpg
test.jpg\nUser logged in as admin

⚙️ Exploitation Steps

  1. Upload file with malicious filename
  2. Check where filename is displayed (page, logs, emails)
  3. If displayed without encoding, XSS/Injection occurs

🛡️ Mitigation

✅ Rename Files: Generate random names, ignore user input
✅ Encode Output: Always encode filename when displaying
✅ Sanitize Input: Remove special characters from filenames
✅ Escape Logs: Properly escape log entries

⚠️ Risk / Impact

Severity: Low to Medium

Impact: XSS, Log Injection, Command Injection
