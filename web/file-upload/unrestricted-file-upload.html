📌 Unrestricted File Upload

🧠 Description

What is Unrestricted File Upload?

The application allows uploading any file type without any validation. Attackers can upload malicious files (PHP, JSP, ASPX, etc.) to execute code on the server.

Why does it occur?

Lack of input validation on file types, missing extension checks, and no content verification.

Security principle violated: Input Validation / Secure Configuration

Real-world analogy: Like a security guard who checks IDs but lets anyone in regardless of what the ID says.

🏷️ Classification

  • Type: Unrestricted File Upload
  • OWASP: A05:2021 – Security Misconfiguration
  • CWE: CWE-434: Unrestricted Upload of File with Dangerous Type

🎯 Attack Surface

  • ✅ Profile picture upload
  • ✅ Document/Attachment upload
  • ✅ Theme/Plugin upload (WordPress, etc.)
  • ✅ CSV/Excel import features
  • ✅ Avatar/Logo upload

🔍 Detection Methodology

1. Static Analysis (Source Code)

// Vulnerable code pattern
move_uploaded_file($_FILES['file']['tmp_name'], "/uploads/" . $_FILES['file']['name']);
// No validation on file type or extension

2. Dynamic Analysis (Burp Suite / Manual)

  • Attempt to upload shell.php
  • Check if upload succeeds
  • Try to access uploaded file

Tools: Burp Suite, OWASP ZAP, custom scripts

⚙️ Exploitation Steps

  1. Identify file upload functionality
  2. Attempt to upload shell.php
  3. If successful, navigate to uploaded file
  4. Execute commands via webshell
  5. Escalate to full server compromise

🛡️ Mitigation

✅ Allowlist Extensions: Only allow safe extensions (jpg, png, gif, pdf, txt)
✅ Validate File Content: Check magic bytes, not just extension
✅ Rename Uploaded Files: Use random names, remove original extension
✅ Store Files Outside Webroot: Prevent direct access
✅ Disable Execution: Configure upload directory to disable script execution
✅ Set Proper Permissions: 644 for files, 755 for directories 
# Secure implementation (PHP)
$allowed = ['jpg', 'png', 'gif', 'pdf'];
$ext = strtolower(pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION));
if (!in_array($ext, $allowed)) {
    die('Invalid file type');
}
$new_name = bin2hex(random_bytes(16)) . '.' . $ext;
move_uploaded_file($_FILES['file']['tmp_name'], "/uploads/" . $new_name);
