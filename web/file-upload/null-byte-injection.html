📌 Null Byte Injection

🧠 Description

What is Null Byte Injection?

Using null byte (%00) to terminate string prematurely, tricking the server about file extension.

Why does it occur?

Vulnerable in older PHP versions (<5.3.4) and C-based file operations where null byte terminates string.

Security principle violated: Input Validation

🏷️ Classification

  • Type: Null Byte Injection
  • CWE: CWE-158: Improper Neutralization of Null Byte

🎯 Attack Surface

  • ✅ PHP versions before 5.3.4
  • ✅ File operations using C functions
  • ✅ Upload endpoints vulnerable to null byte termination

💣 Basic Payloads

shell.php%00.jpg
shell.php%00.png
shell.php%00.gif

# URL encoded
shell.php%00.jpg
shell.php%252E%252E

🧪 Test Cases (STRICT)

StepActionExpectedActual
1Upload shell.php%00.jpgRejectedUpload successful

⚙️ Exploitation Steps

  1. Intercept upload request in Burp Suite
  2. Add %00 null byte before allowed extension
  3. Server truncates filename at null byte
  4. File saved as shell.php, bypasses extension check

🛡️ Mitigation

✅ Update PHP Version: Use PHP 5.3.4+ which patches null byte issue
✅ Sanitize Filenames: Remove null bytes before processing
✅ Use Random Names: Don't use user-supplied filenames

⚠️ Risk / Impact

Severity: High (depends on PHP version)

Impact: Remote Code Execution on vulnerable versions
