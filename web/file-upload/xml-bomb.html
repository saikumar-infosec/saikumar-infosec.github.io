📌 XML Bomb via Upload

🧠 Description

What is XML Bomb?

Uploading malicious XML files to cause Denial of Service via entity expansion (Billion Laughs attack).

Why does it occur?

XML parsers process external entities and recursive expansions without limits.

Security principle violated: Input Validation

🏷️ Classification

  • Type: Denial of Service
  • CWE: CWE-776: XML Entity Expansion

🎯 Attack Surface

  • ✅ XML file upload (configuration, data import)
  • ✅ SOAP web services
  • ✅ RSS feed imports

💣 Basic Payloads

<!DOCTYPE lolz [
  <!ENTITY lol "lol">
  <!ENTITY lol2 "&lol;&lol;">
  <!ENTITY lol3 "&lol2;&lol2;">
  <!ENTITY lol4 "&lol3;&lol3;">
  <!ENTITY lol5 "&lol4;&lol4;">
]>
<lolz>&lol5;</lolz>

⚙️ Exploitation Steps

  1. Create XML file with entity expansion
  2. Upload to application that parses XML
  3. Parser expands entities exponentially
  4. Memory exhaustion causes DoS

🛡️ Mitigation

✅ Disable External Entities: Configure parser to reject DTDs
✅ Set Entity Limits: Limit entity expansion depth
✅ Use Secure Parsers: JSON over XML when possible

⚠️ Risk / Impact

Severity: Medium (DoS)

Impact: Denial of Service, Memory Exhaustion
