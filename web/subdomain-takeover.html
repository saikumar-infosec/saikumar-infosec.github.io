High Severity

📌 Subdomain Takeover

🧠 Description

A DNS CNAME record points to an external service (e.g., AWS S3, GitHub Pages, Heroku, Azure) that is no longer in use. An attacker claims the resource and hosts malicious content on the victim’s subdomain, enabling phishing, XSS, or session theft.

🔍 Detection

# Check for dangling CNAME
dig CNAME sub.example.com
# If response points to unclaimed bucket, it's vulnerable

🚀 Exploitation

  • Identify unclaimed AWS S3 bucket from CNAME.
  • Create bucket with same name and upload an HTML file (e.g., phishing page).
  • Wait for victims to visit the subdomain.

🛡️ Mitigation

✅ Regularly audit DNS records.
✅ Remove unused CNAME records.
✅ Use AWS CloudFront or ALB with custom domain validation.
✅ Monitor for orphaned resources.
