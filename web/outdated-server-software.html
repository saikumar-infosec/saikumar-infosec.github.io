📌 Outdated Web Server Software

🧠 Description

What is the vulnerability?

The web server (Apache, Nginx, IIS, Tomcat) runs an outdated version with known vulnerabilities, allowing attackers to bypass security controls, cause DoS, or execute arbitrary code.

Why does it occur?

Lack of patch management, legacy systems, or fear of downtime.

Security principle violated: Security Misconfiguration / Patch Management

Real-world analogy: A fortress with a crumbling wall – attackers know exactly where to strike.

🏷️ Classification

  • Type: Vulnerable Components
  • OWASP: A06:2021 – Vulnerable and Outdated Components
  • CWE: CWE-1104: Use of Unmaintained Third Party Components

🎯 Attack Surface

  • ✅ HTTP response headers (Server: Apache/2.2.22)
  • ✅ Default error pages
  • ✅ Static file access revealing version (e.g., /icons/, /docs/)
  • ✅ Exposed server status pages

🔍 Detection Methodology

# Check Server header
curl -I https://target.com | grep -i server

# Nmap script for server version
nmap -sV --script http-server-header -p 80,443 target.com

# Use whatweb
whatweb https://target.com

💣 Basic Exploit (Example: Apache 2.2.22 CVE-2012-0053)

# Apache 2.2.22 (CVE-2012-0053) – information disclosure
curl -H "Host: $'\x80\x00\x00\x08\x00\x00\x00\x01'" http://target.com

🚀 Advanced Exploitation

# Searchsploit for known vulnerabilities
searchsploit apache 2.2.22

# Metasploit module for Apache Tomcat
use exploit/multi/http/tomcat_mgr_deploy

🧪 Test Cases (STRICT)

StepActionExpectedActual

⚙️ Exploitation Steps

  1. Identify web server version from headers or error pages.
  2. Search for public exploits (Exploit‑DB, CVE details).
  3. Execute exploit to gain RCE or information disclosure.
  4. Use compromised server as pivot point.

🛡️ Mitigation

✅ Keep web server software updated to latest stable version.
✅ Remove version headers (ServerTokens Prod in Apache, server_tokens off in Nginx).
✅ Use a WAF to virtual‑patch known vulnerabilities.
✅ Automate patch management.
✅ Harden server configuration (remove unnecessary modules).

🔓 Mitigation Bypass

  • If version is hidden but outdated, attackers may still fingerprint via behavior or known exploit patterns.
  • Backported patches may not cover all vulnerabilities.

⚠️ Risk / Impact

Severity: High

Impact: Remote Code Execution, information disclosure, denial of service, full server compromise.

📎 Proof of Concept (PoC)

# Simple version check
curl -I https://target.com | grep -i "server"

# If output shows "Server: Apache/2.2.22", it's vulnerable to multiple CVEs
