High Severity

📌 Outdated Web Server Software

🧠 Description

What is the vulnerability?

The web server (Apache, Nginx, IIS, Tomcat) runs an outdated version with known vulnerabilities, allowing attackers to bypass security controls, cause DoS, or execute arbitrary code.

Why does it occur?

Lack of patch management, legacy systems, or fear of downtime.

🏷️ Classification

  • OWASP: A06:2021 – Vulnerable and Outdated Components
  • CWE: CWE-1104

🎯 Attack Surface

  • ✅ HTTP response headers (Server: Apache/2.2.22)
  • ✅ Default error pages
  • ✅ Static file access revealing version

🔍 Detection Methodology

curl -I https://target.com | grep -i server
nmap -sV --script http-server-header -p 80,443 target.com

💣 Basic Exploit

# Example for Apache 2.2.22 (CVE-2012-0053)
curl -H "Host: $'\x80\x00\x00\x08\x00\x00\x00\x01'" http://target.com

🛡️ Mitigation

✅ Keep web server software updated to latest stable version.
✅ Remove version headers (ServerTokens Prod in Apache, server_tokens off in Nginx).
✅ Use a WAF to virtual‑patch known vulnerabilities.
✅ Automate patch management.
