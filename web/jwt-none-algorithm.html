Critical Severity

📌 JWT None Algorithm Attack

🧠 Description

The JWT standard includes a special algorithm none intended for debugging or internal trusted systems. If a server accepts tokens with alg: none, it does not verify any signature. An attacker can forge a valid token for any user, including admin, by modifying the payload and setting the algorithm to none.

💣 Attack Steps

  1. Capture a valid JWT from the application.
  2. Decode it at jwt.io.
  3. In the header, change "alg": "HS256" to "alg": "none".
  4. In the payload, change "role": "user" to "role": "admin".
  5. Remove the signature (leave the third part empty).
  6. Send the crafted token to a privileged endpoint.
  7. If the server accepts it, the vulnerability is confirmed.

🛡️ Mitigation

✅ Never accept the none algorithm in production
✅ Whitelist allowed algorithms (e.g., only HS256 and RS256)
✅ Use a JWT library that rejects none by default
