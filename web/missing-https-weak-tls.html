📌 Missing HTTPS or Weak TLS Configuration

🧠 Description

What is the vulnerability?

The application does not enforce HTTPS or uses outdated TLS versions (TLS 1.0, TLS 1.1) or weak cipher suites. Attackers on the same network can intercept, modify, or downgrade connections (MITM), stealing credentials, session tokens, or other sensitive data.

Why does it occur?

Legacy configurations, lack of proper SSL/TLS management, or incomplete migration to HTTPS.

Security principle violated: Cryptographic Failure / Transport Layer Security

Real-world analogy: Sending a postcard instead of a sealed envelope – anyone handling it can read the contents.

🏷️ Classification

  • Type: Cryptographic Failure
  • OWASP: A02:2021 – Cryptographic Failures
  • CWE: CWE-319: Cleartext Transmission of Sensitive Information

🎯 Attack Surface

  • ✅ Login pages served over HTTP
  • ✅ Mixed content (HTTPS page loads HTTP resources)
  • ✅ Absence of HSTS header
  • ✅ Weak TLS versions/ciphers (TLS 1.0, TLS 1.1, RC4, DES)

🔍 Detection Methodology

  • Check URL bar – is it `http://` or `https://`?
  • Use `curl -I https://target.com` to check HSTS header.
  • Use SSL Labs (ssllabs.com) to test TLS configuration.
  • Use `nmap --script ssl-enum-ciphers -p 443 target.com`.

💣 Basic Payloads (MITM Attack)

# ARP spoofing to intercept traffic
sudo arpspoof -t 192.168.1.10 192.168.1.1

🚀 Advanced Exploitation

# sslstrip to downgrade HTTPS to HTTP
sudo sslstrip -l 8080

⚙️ Exploitation Steps

  1. Position yourself on the same network as the victim.
  2. Perform ARP spoofing to become man-in-the-middle.
  3. If HTTPS is not enforced, capture plaintext credentials.
  4. If weak TLS is supported, attempt protocol downgrade attacks.

🛡️ Mitigation

✅ Enforce HTTPS everywhere (HSTS with `preload`).
✅ Redirect all HTTP traffic to HTTPS (301).
✅ Disable TLS 1.0 and 1.1, use TLS 1.2+.
✅ Use strong cipher suites (e.g., ECDHE-RSA-AES256-GCM-SHA384).
✅ Avoid mixed content.
✅ Use tools like SSL Labs to audit configuration.

🔓 Mitigation Bypass

  • If HSTS is missing, an attacker can perform SSL stripping.
  • If TLS 1.0 is still enabled, attackers can use POODLE or other attacks.

⚠️ Risk / Impact

Severity: High

Impact: Credential theft, session hijacking, data modification.

📎 Proof of Concept (PoC)

# Check HSTS
curl -sI https://target.com | grep -i "strict-transport-security"

# Check supported TLS versions
nmap --script ssl-enum-ciphers -p 443 target.com
