📌 Log Injection (Log Forging)

🧠 Description

What is the vulnerability?

Attackers inject newline characters or malicious data into log entries, forging fake log lines, hiding their actions, or exploiting log viewers (e.g., JavaScript injection in log viewers).

Why does it occur?

Logs are constructed by concatenating user‑controlled input without sanitising CR/LF characters.

Security principle violated: Input Validation / Log Integrity

Real-world analogy: Adding fake entries to a police logbook to mislead investigators.

🏷️ Classification

  • Type: Log Forging
  • OWASP: A09:2021 – Security Logging and Monitoring Failures
  • CWE: CWE-117: Improper Output Neutralization for Logs

🎯 Attack Surface

  • ✅ User‑Agent header
  • ✅ Username/login fields
  • ✅ Search queries
  • ✅ Any parameter that gets written to logs

🔍 Detection Methodology

  • Submit input containing `%0a`, `%0d`, `\r`, `\n` and check if logs contain injected lines.
  • Attempt to inject JavaScript (``) and view logs in a web‑based log viewer.

🚀 Advanced Exploitation

# Hide malicious activity by injecting fake successful login after failure
username=admin%0a[INFO] Login successful from 127.0.0.1%0a&password=wrong

# If logs are used for automated parsing, break the log format
username=admin%0a%0aERROR -- Corrupted log entry%0a&password=test
