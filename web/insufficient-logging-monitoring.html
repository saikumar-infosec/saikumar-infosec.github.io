Medium Severity

📌 Insufficient Logging & Monitoring

🧠 Description

The application does not log critical events (failed logins, privilege changes, data access) or lacks real‑time monitoring. Attackers can brute force, escalate privileges, or exfiltrate data without detection.

🔍 Detection

  • Attempt multiple failed logins – check if any log entry or alert appears.
  • Perform a high‑risk action (e.g., privilege escalation) and see if it's logged.

⚙️ Exploitation Impact

Attackers can operate for weeks or months without being noticed, leading to prolonged data theft or backdoor persistence.

🛡️ Mitigation

✅ Log all authentication attempts, privilege changes, and sensitive data access.
✅ Use a SIEM or centralized logging.
✅ Set up alerts for suspicious patterns (multiple failures, unusual locations).
✅ Regularly review logs.
