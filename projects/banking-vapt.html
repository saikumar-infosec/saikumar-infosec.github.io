Banking Application VAPT — Case Study

Full-scope assessment for a large banking mobile + web platform.

Scope

  • External web application (customer portal)
  • Internal APIs (auth & transaction)
  • Mobile app (Android & iOS)

Attack Surface & Methodology

Reconnaissance → Auth testing → Input validation → API business logic testing → Mobile reverse engineering → Privilege escalation checks.

Tools Used

Burp Suite Pro, ZAP, SQLMap, MobSF, Frida, JADX, Nmap

Key Findings

  1. IDOR in transaction API: Allowed unauthorized account fund view. PoC created and fixed.
  2. JWT signature misconfiguration: Weak verification allowing token tampering.
  3. Mobile insecure storage: Sensitive tokens in SharedPreferences.

Business Impact

Potential data exposure and unauthorized transaction risk — high severity. Recommended immediate fixes reduced attack surface and risk of exploitation.

Remediation

  • Implement proper access control checks on server side.
  • Enforce JWT verification and rotate keys.
  • Encrypt sensitive data and remove local tokens; use secure storage.
